## ขั้นตอนที่ 1: วิเคราะห์ Persona (User Persona Analysis)

ระบบร้านหนังสือออนไลน์มีกลุ่มผู้ใช้งานหลัก **3 กลุ่ม** ดังนี้

| # | Persona | ชื่อกลุ่ม | บทบาทและความรับผิดชอบ |
|---|---------|-----------|----------------------|
| 1 | 👤 **Customer** | ลูกค้า / ผู้ซื้อหนังสือ | ผู้ใช้งานทั่วไปที่เข้าสู่ระบบ Storefront เพื่อค้นหาหนังสือ เพิ่มสินค้าลงตะกร้า ชำระเงิน และติดตามสถานะคำสั่งซื้อ รวมถึงจัดการข้อมูลโปรไฟล์ของตนเอง |
| 2 | 🔧 **Staff** | พนักงาน / เจ้าหน้าที่คลังสินค้า | ผู้ใช้งานที่มีสิทธิ์เข้าถึง Staff Portal เพื่อตรวจสอบและจัดการสต็อกหนังสือ (Inventory Management) อัปเดตจำนวนสินค้าคงเหลือ และตรวจสอบสถานะออเดอร์ |
| 3 | 👑 **Admin** | ผู้ดูแลระบบ | ผู้ใช้งานที่มีสิทธิ์สูงสุด เข้าถึง Admin Portal เพื่อจัดการข้อมูลหนังสือ (เพิ่ม/แก้ไข/ลบ), จัดการหมวดหมู่, จัดการบัญชีผู้ใช้ทั้งหมด และดูภาพรวมของระบบ |

### 4.3 สรุปผลการประเมิน UAT (UAT Evaluation Summary)

จากการดำเนินการทดสอบ User Acceptance Testing สำหรับ **ระบบร้านหนังสือออนไลน์ (Online Bookstore) v1.0.0** จำนวนทั้งสิ้น **11 Test Cases** ครอบคลุมผู้ใช้งาน 3 กลุ่ม (Customer, Staff, Admin) ปัจจุบัน **ผ่านการทดสอบครบทั้ง 11 รายการ** คิดเป็น **อัตราการผ่านการทดสอบโดยรวม 100%**

ฟังก์ชันหลักของระบบที่ทำงานได้อย่างถูกต้องและครบถ้วน ได้แก่ การลงทะเบียนบัญชีผู้ใช้, การค้นหาและดูรายละเอียดหนังสือ, การเพิ่มสินค้าลงตะกร้าและ Checkout, การติดตามและดูรายละเอียดคำสั่งซื้อ, การเข้าสู่ระบบ Staff Portal พร้อมระบบ Role-based Access Control, การอัปเดตสต็อกสินค้า (Persist ลงฐานข้อมูล), และการจัดการข้อมูลหนังสือ (เพิ่ม/แก้ไข/ลบ) รวมถึงการจัดการ Role ผู้ใช้โดย Admin

ทีมพัฒนาได้แก้ไขปัญหาทั้ง 3 รายการเรียบร้อยแล้ว:

- **ISS-001 (Resolved):** สร้างหน้า `/orders/[id]` และเพิ่ม Link "รายละเอียด" ใน `OrdersView.tsx` สำหรับให้ลูกค้าดูรายการย่อยของแต่ละคำสั่งซื้อ
- **ISS-002 (Resolved):** `InventoryView.tsx` ส่ง `PUT /api/books/[id]` จริง พร้อม Confirmation Modal ก่อนบันทึก ข้อมูลสต็อกถูก Persist ลงฐานข้อมูลอย่างถูกต้อง
- **ISS-003 (Resolved):** `AdminUsersView.tsx` มี Edit Modal พร้อม Role dropdown (customer/staff/admin) และ Confirmation Modal ก่อนบันทึก

**🎉 UAT Result: PASS — ระบบพร้อม Deploy สู่ Production** แนะนำให้ดำเนินการ Regression Testing รอบสุดท้ายก่อน Deploy เพื่อความมั่นใจสูงสุด
