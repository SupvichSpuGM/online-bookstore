ระบบร้านหนังสือออนไลน์ (Online Bookstore) — ทดสอบครอบคลุมทุก Persona และฟังก์ชันหลัก

🧪 จำนวน Test Cases: 11 รายการ

STEP 01

การวิเคราะห์ Persona (User Persona Analysis)

👤
Customer · ลูกค้า
Storefront User

ผู้ใช้งานทั่วไปที่เข้าสู่ระบบ Storefront เพื่อค้นหาหนังสือ เพิ่มสินค้าลงตะกร้า ชำระเงิน และติดตามสถานะคำสั่งซื้อ รวมถึงจัดการข้อมูลโปรไฟล์ของตนเอง

🔧
Staff · พนักงาน
Inventory Manager

ผู้ใช้งานที่มีสิทธิ์เข้าถึง Staff Portal เพื่อตรวจสอบและจัดการสต็อกหนังสือ (Inventory Management) อัปเดตจำนวนสินค้าคงเหลือ และตรวจสอบสถานะออเดอร์

👑
Admin · ผู้ดูแลระบบ
System Administrator

ผู้ใช้งานที่มีสิทธิ์สูงสุด เข้าถึง Admin Portal เพื่อจัดการข้อมูลหนังสือ (เพิ่ม/แก้ไข/ลบ), จัดการหมวดหมู่, จัดการบัญชีผู้ใช้ทั้งหมด และดูภาพรวมของระบบ

🎉 UAT Final Evaluation Summary — Online Bookstore v1.0.0

จากการดำเนินการทดสอบ User Acceptance Testing สำหรับระบบร้านหนังสือออนไลน์ v1.0.0 จำนวนทั้งสิ้น 11 Test Cases ครอบคลุมผู้ใช้งาน 3 กลุ่ม (Customer, Staff, Admin) ปัจจุบัน ผ่านการทดสอบครบทั้ง 11 รายการ คิดเป็น อัตราการผ่าน 100%

ทีมพัฒนาได้แก้ไขปัญหาทั้ง 3 รายการเรียบร้อยแล้ว ได้แก่ ISS-001 (Order Detail Page), ISS-002 (Stock Persist via API), และ ISS-003 (Role Management Modal) ฟังก์ชันหลักทั้งหมดทำงานได้ถูกต้องและครบถ้วน

✅ UAT Result: PASS — พร้อม Deploy สู่ Production
ระบบผ่าน UAT ครบ 11/11 Test Cases แนะนำให้ดำเนินการ Regression Testing รอบสุดท้ายก่อน Deploy เพื่อความมั่นใจสูงสุด
